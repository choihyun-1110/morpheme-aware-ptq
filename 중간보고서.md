# 형태소 다양성 기반 GPTQ Calibration 데이터 선택이 한국어 LLM 양자화 품질에 미치는 영향

**제출일:** 2026-04-17
**연구자:** 최현
**연구 기간:** 2026-02 ~ 2026-04 (진행 중)

---

## 1. 배경 및 연구 동기

### 1.1 한국어 LLM의 확산과 경량화 필요성

SOLAR-10.7B, HyperCLOVA X, EXAONE 등 한국어 특화 대규모 언어 모델(LLM)의 등장으로, 이를 실제 서비스에 배포하기 위한 경량화 기술의 필요성이 크게 증가하고 있다. 파라미터가 수십억 개에 달하는 이러한 모델을 일반적인 GPU 환경에서 운용하려면 메모리 절감과 추론 속도 향상이 필수적이다.

사후 훈련 양자화(Post-Training Quantization, PTQ)는 모델을 재학습하지 않고 가중치를 저정밀도(4-bit, 8-bit)로 변환하는 기법으로, 실용적인 경량화 방법으로 주목받고 있다. 그러나 PTQ는 양자화 과정에서 불가피한 정밀도 손실이 발생하며, 이를 보정하기 위해 소량의 **calibration 데이터**를 활용한다.

### 1.2 Calibration 데이터의 중요성

GPTQ(Frantar et al., 2022) 등 최신 PTQ 방법론에서 calibration 데이터는 레이어별 가중치 근사 오차를 최소화하는 데 사용된다. 기존 연구들은 대부분 영어 텍스트(Wikitext-2 등)를 calibration 데이터로 사용하지만, 한국어 모델에서도 동일한 방식이 최적인지에 대한 연구는 부재하다.

**핵심 질문:** *한국어 LLM을 양자화할 때, calibration 데이터의 언어와 언어적 다양성이 PTQ 품질에 영향을 미치는가?*

## 2. 관련 연구

### 2.1 GPTQ (Frantar et al., 2022)

GPTQ는 OBQ(Optimal Brain Quantization)를 확장한 방법으로, Hessian 행렬 근사를 이용하여 레이어별 가중치를 4-bit로 양자화한다. 핵심 아이디어는 양자화 오차를 아직 양자화되지 않은 가중치로 보상(error compensation)하는 것이다.

Calibration 데이터는 이 Hessian 행렬을 계산하는 데 사용된다. 즉, calibration 데이터가 입력 activation 분포를 대표하지 못하면 Hessian 근사가 편향되고, 가중치 보정 방향이 왜곡될 수 있다.

### 2.2 AWQ (Lin et al., 2023)

AWQ(Activation-aware Weight Quantization)는 가중치 채널의 saliency를 activation 크기 기반으로 측정하여 중요한 채널에 더 높은 정밀도를 할당한다. AWQ 역시 calibration 데이터로부터 activation 통계를 수집하며, 이때 데이터의 분포가 모델 성능에 직접적인 영향을 준다고 보고한다.

### 2.3 SmoothQuant (Xiao et al., 2022)

SmoothQuant는 LLM 추론 시 나타나는 outlier activation 문제를 해결하기 위해 activation과 가중치 사이에 스케일 변환을 도입한다. Outlier가 특정 채널에 집중되는 현상은 calibration 데이터의 다양성과 관련이 있으며, 다양한 입력 패턴을 포함한 calibration이 outlier 추정을 개선할 수 있다.

### 2.4 Calibration 데이터의 영향

Sheng et al. (2023)은 LLM serving에서 calibration 데이터 선택이 KV cache 압축 품질에 영향을 준다고 보고한다. Dettmers et al. (2023, QLoRA)은 calibration 데이터의 도메인 일치가 fine-tuned 모델의 양자화 품질에 중요하다는 점을 간접적으로 시사한다. 그러나 **언어 자체의 차이**가 calibration 효과에 미치는 영향을 직접 실험한 연구는 현재까지 발견되지 않았다.

## 3. 연구 질문 및 가설

### 3.1 주요 연구 질문

1. **언어 영향**: 영어 calibration과 한국어 calibration 중 한국어 LLM 양자화에 어느 것이 더 유리한가?
2. **다양성 영향**: 동일 언어 내에서 형태소 다양성이 높은 calibration이 랜덤 샘플보다 성능 보존에 유리한가?
3. **언어 정렬 원리**: "사전학습 주 언어 = calibration 언어"일 때 GPTQ 품질이 최적화되는 일반 원리가 존재하는가?

### 3.2 가설 발전 과정

**초기 가설 (Sprint 1):**
형태소 다양성 기반 한국어 calibration set이 랜덤 영어/한국어보다 GPTQ 4-bit 품질 보존에 유리하다.

**정제 가설 (Sprint 2 이후):**
단순히 형태소 다양성이 좋은 게 아니라, **사전학습 주 언어와 calibration 언어가 일치할 때** 성능 보존에 유리하다. 이때 비한국어 subword 과다가 한국어 처리 레이어의 Hessian 근사를 왜곡하는 메커니즘이 작동한다.

**현재 상태 (Sprint 3):**
Qwen2-7B 실험에서 가설이 단순 검증되지 않고 새로운 해석 갈림길이 발생했다. "평가 언어 = calibration 언어" 혹은 "형태소 다양성의 범언어적 효과" 두 가지 설명이 경쟁 중이며, 추가 실험으로 규명이 필요하다.

## 4. 연구 방법론 및 아이디어

### 4.1 형태소 다양성 기반 Calibration Set 선택 알고리즘

핵심 아이디어는 calibration 문장을 **형태소 다양성(morpheme diversity)**을 최대화하는 방향으로 greedy하게 선택하는 것이다.

**알고리즘 개요:**

```
입력: 후보 문장 풀 (위키피디아 등), 목표 문장 수 N
출력: 선택된 N개 문장 (calibration set C)

1. 현재 선택된 집합의 unique 형태소 집합 M 초기화
2. 각 후보 문장 s에 대해:
   - 형태소 분석 수행 (한국어: Mecab/Kiwi, 중국어: jieba)
   - new_morphemes(s) = s의 형태소 ∖ M (신규 형태소 수)
   - SFS(s) = |new_morphemes(s)| / len(s) (단위 길이당 신규 형태소)
3. SFS 최대 문장을 선택, M 업데이트
4. N개 문장이 될 때까지 반복
```

**C_v3 추가 개선 사항:**
- `min_ko_ratio=0.7`: 한국어 모델용으로 한국어 형태소 비율이 70% 미만인 문장 제외
- 비한국어 subword 토큰 과다 문제 방지 → GPTQ Hessian 근사 왜곡 억제

**중국어 확장 (C_zh):**
- `jieba` 라이브러리로 중국어 형태소 분석
- `wikimedia/wikipedia 20231101.zh` 데이터 활용
- 동일 greedy 선택 알고리즘 적용

### 4.2 평가 체계

| 평가셋 | 언어 | 태스크 수 | 용도 |
|--------|------|-----------|------|
| KoBEST | 한국어 | 5 (boolq/copa/hellaswag/sentineg/wic) | 한국어 이해 능력 |
| kmmlu | 한국어 | 다수 과목 | 한국어 지식 |
| C-Eval | 중국어 | 다수 과목 | 중국어 모델 검증 (예정) |

## 5. 예비 실험 결과

### 5.1 SOLAR-10.7B-Instruct 실험 (Sprint 2 — 완료)

**실험 설정:**
- 모델: SOLAR-10.7B-Instruct-v1.0 (한국어 주요 사전학습)
- 양자화: GPTQ 4-bit, group_size=128, desc_act=True
- Calibration 조건: A(영어 랜덤), B(한국어 랜덤), C_v3(한국어 형태소 다양성)

**KoBEST 결과:**

| 조건 | KoBEST 평균 | FP16 보존율 | 하락폭 |
|------|------------|------------|--------|
| FP16 베이스라인 | 0.6523 | 100% | — |
| **C_v3 (한국어 다양성)** | **0.6356** | **97.4%** | **-0.017** |
| B (한국어 랜덤) | 0.6176 | 94.7% | -0.035 |
| A (영어 랜덤) | 0.5981 | 91.7% | -0.054 |

- C_v3가 FP16 대비 **97.4% 성능 보존** — B(94.7%) 대비 2.7%p 우위
- 재현성 확인: Run1=0.6354, Run2=0.6356 (차이 0.0002)
- **kmmlu에서도 C_v3 > B > A 순서 유지** (C_v3=0.3747, B=0.3731, A=0.3677)

**핵심 메커니즘 발견:**

| 지표 | B (한국어 랜덤) | C_v3 (한국어 다양성) |
|------|----------------|---------------------|
| 고유 subword 커버리지 | 527 | 570 |
| 비한국어 subword 비율 | 낮음 | **낮음** (min_ko_ratio=0.7 적용) |
| 평균 SFS | 2.30 | 6.57 |

원본 C(calibration)에서는 비한국어 subword가 846개로 과다하여 B보다 성능이 낮았다. C_v3는 비한국어 subword를 570개로 줄이면서 형태소 다양성은 유지하여 성능 회복에 성공했다.

**해석:** 비한국어 subword 토큰 과다 → GPTQ Hessian 행렬 편향 → 한국어 처리 가중치 근사 왜곡 → 성능 저하

### 5.2 다모델 검증 (Sprint 2 후반 — 완료)

동일 calibration set(A/B/C_v3)을 한국어 관련 다른 모델에 적용:

| 모델 | 사전학습 주 언어 | A | B | C_v3 | 최우수 |
|------|----------------|---|---|------|--------|
| SOLAR-10.7B | 한국어 | 0.5981 | 0.6176 | **0.6356** | C_v3 |
| Llama3-Ko-8B | 영어 (LLaMA-3 기반) | **0.5758** | 0.5608 | 0.5650 | A |
| EEVE-10B | 한국어 (SOLAR 기반 SFT) | 0.7172 | **0.7595** | 0.7514 | B |

**관찰:**
- Llama3-Ko: 베이스 모델이 영어 LLaMA-3 → 영어 calibration이 최선 → "사전학습 언어 = calibration 언어" 가설 지지
- EEVE: SOLAR 기반이나 C_v3보다 B(한국어 랜덤)가 미소 우위 → 추가 분석 필요

### 5.3 Qwen2-7B 실험 (Sprint 3 — 완료, 부분 해석 보류)

**가설:** Qwen2-7B(중국어 사전학습)에서 C_zh > C_v3 > A ("사전학습 언어 정렬 원리" 검증)

**KoBEST 결과:**

| 조건 | KoBEST 평균 |
|------|------------|
| C_v3 (한국어 다양성) | **0.6375** |
| A (영어 랜덤) | 0.6238 |
| C_zh (중국어 다양성) | 0.6063 |

**순위: C_v3 > A > C_zh — 가설 기각**

중국어 모델임에도 중국어 calibration이 최저. 두 가지 해석이 경쟁:
1. **평가 언어 정렬 가설**: KoBEST가 한국어 평가 → 한국어 calibration이 한국어 능력을 더 잘 보존
2. **형태소 다양성 범언어적 효과 가설**: C_v3의 greedy 알고리즘 자체가 더 좋은 activation 다양성을 유도

→ **C-Eval/CMMLU(중국어 벤치마크)로 재평가 필요**: Qwen2의 중국어 능력 측면에서 C_zh가 더 우수한지 확인

### 5.4 vLLM 서빙 벤치마크 (Sprint 3 — 완료)

GPTQ 양자화 모델의 실제 서빙 성능을 vLLM으로 측정:

| 지표 | SOLAR FP16 | SOLAR GPTQ C_v3 | 변화 |
|------|-----------|-----------------|------|
| 모델 파일 크기 | 20GB | 5.6GB | **-72%** |
| Throughput | 482.87 tok/s | 502.25 tok/s | +4% |
| 첫 토큰 지연 | 465.7ms | 443.5ms | -5% |

GPTQ 4-bit 양자화로 **72% VRAM 절감**을 달성하면서도 throughput과 latency가 소폭 개선되었다. 이는 GPTQ 양자화가 품질 보존뿐 아니라 서빙 효율성 측면에서도 실용적임을 보여준다.

## 6. 현재 연구 상태 요약 및 해석

### 6.1 확립된 발견

1. **SOLAR에서 형태소 다양성 calibration 유효**: C_v3가 FP16 보존율 97.4%로 영어(91.7%), 한국어 랜덤(94.7%) 대비 우위. 재현성 확인.

2. **비한국어 subword 과다가 핵심 오류 원인**: 단순히 다양성이 높은 것이 아니라, 목표 언어가 아닌 subword 토큰을 제거했을 때 성능이 회복됨.

3. **언어-모델 정렬 일부 지지**: Llama3-Ko에서 영어 calibration이 최선 → 사전학습 언어와 calibration 언어 일치 원리 부분 지지.

### 6.2 미해결 질문

1. **Qwen2 해석 갈림길**: "평가 언어 정렬"인가 "형태소 다양성 범언어적 효과"인가 → C-Eval 재평가로 규명 필요

2. **EEVE B > C_v3 원인**: EEVE tokenizer와 C_v3 calibration(SOLAR tokenizer 기준 생성)의 불일치 가능성

3. **통계적 유의성**: B vs C_v3 차이(0.018)에 대한 신뢰구간 미계산

1. **다중 런 통계 검정**: C_v3 vs B 3회 이상 반복 → 95% 신뢰구간 계산
2. **Hessian 근사 오차 분석**: calibration 언어별 레이어별 양자화 오차 측정
3. **논문 초고 작성**: 연구 결과를 학술 논문 형태로 정리

### 7.3 최종 연구 주장 방향

Qwen2 C-Eval 결과에 따라 두 가지 방향으로 분기:

**Case A: C_zh > C_v3 (C-Eval 기준)**
→ "사전학습 주 언어 = calibration 언어일 때 해당 언어 능력이 최적 보존"
→ 언어 정렬 원리가 평가 언어와 무관하게 성립

**Case B: C_v3 ≥ C_zh (C-Eval 기준)**
→ "형태소 다양성 알고리즘 자체가 언어를 초월하는 activation 다양성을 유도"
→ 논문 주장을 언어 독립적인 calibration 최적화 알고리즘으로 재구성

두 경우 모두 **형태소 다양성 greedy 선택 알고리즘**의 기여는 유지되며, 언어 선택 전략만 달라진다.

## 8. 결론

본 연구는 한국어 LLM의 GPTQ 4-bit 양자화 시 calibration 데이터의 언어적 특성이 성능 보존에 유의미한 영향을 미친다는 것을 실증하고 있다. SOLAR-10.7B에서 형태소 다양성 기반 한국어 calibration(C_v3)이 FP16 대비 97.4% 성능을 보존하여, 영어 랜덤(91.7%) 대비 5.7%p 우위를 보였다.

핵심 메커니즘은 비한국어 subword 토큰의 과다가 GPTQ의 Hessian 행렬 추정을 편향시켜 한국어 처리 가중치 근사를 왜곡한다는 것으로 추정된다. 다모델 실험에서 사전학습 언어와 calibration 언어의 일치가 성능 보존에 유리하다는 패턴이 관찰되었으나, Qwen2 실험에서 새로운 해석 갈림길이 발생하여 추가 실험이 진행 중이다.

이 연구는 한국어/다국어 LLM의 실용적 배포를 위한 PTQ 최적화에 구체적인 방법론적 기여를 제공하며, calibration 데이터 선택이 간과되어 온 중요한 변수임을 보여준다.
