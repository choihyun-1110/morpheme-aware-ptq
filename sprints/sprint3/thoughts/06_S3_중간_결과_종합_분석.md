## 3. 가설 검토

### 원 가설: "사전학습 언어 = calibration 언어 → 최우수 GPTQ"

**결과:**
- SOLAR: ✅ 한국어 사전학습 + C_v3(한국어) = 최우수 ← 지지
- Llama3-Ko: ✅ 영어 기반 + A(영어) = 최우수 ← 지지
- Qwen2: ❌ 중국어 사전학습 + C_zh(중국어) = **최하** ← 반증
- EEVE: ⚠️ 한국어 SFT + B(한국어) ≈ C_v3 ← 약한 지지

**판정: 가설 부분 지지, 일반 원리로 성립 불가**

## 4. 대안 가설 및 현재 최선 설명

### 가설 A: 형태소 다양성이 핵심

**C_v3의 공통점:**
- 높은 SFS (Subword Fertility Score)
- 비한국어 subword 최소화 (SOLAR tokenizer 기준)
- 다양한 형태소 구조 포함

**설명력:**
- SOLAR에서 C_v3 > B: ✅ B보다 SFS 높음, 비한국어 subword 적음
- Qwen2에서 C_v3 > C_zh: ✅ C_zh는 avg_SFS=2.29로 낮음 (C_v3는 더 높음)
- Qwen2에서 C_v3 > A: ✅ A는 영어 단어 위주로 Qwen2 한국어 처리 커버 못함
- Llama3-Ko에서 A > C_v3: ⚠️ Llama3의 tokenizer가 영어 최적화 → A의 SFS가 실질적으로 높을 수 있음

**한계:**
- "SOLAR tokenizer 기준 C_v3"가 다른 모델(Qwen2, EEVE)에 얼마나 적절한가?
- EXAONE35 결과가 나와야 패턴 확인 가능

### 가설 B: Tokenizer 효율성이 매개 변수

**핵심 아이디어:**
각 모델의 tokenizer로 calibration text를 인코딩할 때 **token 효율성**이 높을수록 GPTQ가 더 좋은 Hessian 추정을 할 수 있다.

- 같은 128문장이라도 tokenizer마다 총 token 수가 다름
- 한국어 텍스트는 모델별로 token 수 차이가 큼

**검증 방법:**
각 조건별 calibration text를 각 모델의 tokenizer로 인코딩하여 평균 token 수 비교.

## 5. EXAONE35 실험의 의의

EXAONE35 (LG AI Research):
- **순수 한국어 + 영어 사전학습**
- SOLAR과 다르게 "한국어 특화 사전학습" (Mistral 기반이 아닌 자체 아키텍처)

**예상:**
- SOLAR 패턴 재현: C_v3 > B > A → "한국어 사전학습 모델에서 C_v3 우위" 일반화
- 다른 패턴: A 또는 B 최우수 → EXAONE의 독특한 한국어 처리 방식

EXAONE35 결과가 SOLAR과 같으면:
→ "한국어 사전학습 모델 + C_v3(한국어 형태소 다양성)" 조합의 일반 원리 강력히 지지

## 6. Phase 2 실험의 의의

Phase 2에서 얻을 수 있는 추가 정보:

### group_size=64 vs 128
- 64가 더 세밀한 양자화 → 일반적으로 품질 향상
- C_v3의 이점이 group_size에 따라 달라지는가?
- 예상: g64에서 C_v3 이점 유지 또는 확대

### SmoothScale + GPTQ
- activation outlier를 줄인 후 GPTQ → Hessian 추정 안정화
- C_v3의 이점이 SmoothScale로 대체 가능한가?
- 예상: SmoothScale은 C_v3 이점과 독립적 (가산적 효과)

### AWQ
- activation magnitude 기반 방법론
- GPTQ에서의 calibration 언어 효과가 AWQ에서도 나타나는가?
- 예상: AWQ는 calibration 언어에 덜 민감할 수 있음 (activation 크기가 언어와 관계없이 높은 채널 보호)

### desc_act=False
- activation 기반 컬럼 재정렬 제거
- C_v3의 channel_cv 우위가 desc_act를 통해 발현된다면 → desc_act=False에서 C_v3 이점 감소
- 이 실험은 **C_v3 효과의 메커니즘** 이해에 중요
