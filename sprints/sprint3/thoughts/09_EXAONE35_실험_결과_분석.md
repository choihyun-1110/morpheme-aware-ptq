- B: 0.904

→ EXAONE35의 한국어 감성 처리 능력은 **한국어 calibration이 없으면 거의 작동하지 않음**

### 2. B와 C_v3가 거의 동점

B: 0.7196 vs C_v3: 0.7164 → 격차 0.0032 (통계적으로 유의하지 않을 수 있음)

태스크별 승패:
- sentineg: C_v3=0.922 > B=0.904 (+0.018)
- wic: C_v3=0.601 > B=0.533 (+0.068) ← C_v3 우위
- boolq: B=0.891 > C_v3=0.804 (+0.087) ← B 우위
- copa: B=0.776 > C_v3=0.759 (+0.017)
- hellaswag: C_v3=0.496 ≈ B=0.494 (동점)

전반적으로 B와 C_v3는 서로 다른 태스크에서 우위를 보여 **통계적으로 동등한 수준**.

## 가설 검토

### 가설 A: 사전학습 언어 정렬

> "사전학습 주 언어 = calibration 언어 → 최우수 GPTQ"

**판정: 강하게 지지**
- EXAONE35(한국어+영어): 한국어 B/C_v3 >> 영어 A ✅
- SOLAR(한국어): C_v3/B >> A ✅
- Llama3-Ko(영어 기반): A >> B/C_v3 ✅
- Qwen2(중국어): C_v3 > A > C_zh ← 예외 (C_zh 최악)
- EEVE(한국어 SFT): B/C_v3 >> A ✅

### 가설 B: 형태소 다양성

> "형태소 다양성 높은 calibration → 최우수 GPTQ"

**판정: 보통 지지**
- EXAONE35에서 B ≈ C_v3 → 형태소 다양성보다는 "한국어 여부"가 더 중요해 보임
- SOLAR에서만 C_v3 > B (C_v3 형태소 다양성 효과 명확)
- Qwen2에서 C_zh 최악 → 단순 "사전학습 언어 일치"가 아니라 다양성도 관련

### 현재 최선 해석

**"사전학습 언어 정렬이 primary 요인, 형태소 다양성은 secondary 요인"**

1. 한국어 사전학습 모델 → 한국어 calibration (B 또는 C_v3) 필수
2. B vs C_v3 차이는 모델마다 다름 (형태소 다양성 효과가 항상 일관되지 않음)
3. SOLAR에서만 C_v3 > B로 명확 → SOLAR tokenizer 기준으로 생성된 C_v3의 이점이 SOLAR에서만 극대화
