## 1. 목적 (Why do we need this?)

우리 연구의 핵심 가설은 **"한국어 형태소의 다양성이 높은 문장들로 Calibration 데이터셋을 구성하면, 양자화 후 활성화(Activation) 분포의 형태를 더 잘 보존하여 성능 저하가 덜 할 것이다"**입니다.

이를 위해서는 대량의 텍스트(예: 위키백과 수십만 문장)를 빠르게 형태소 분석하고, '문장 내 형태소 종류 수'를 계량화할 수 있는 강력한 한국어 형태소 분석기가 필요합니다.

### 📌 선택 기준
1. **속도:** 백만 단위 문장을 순회하며 다양성 지수를 계산해야 하므로 연산 속도가 매우 중요.
2. **설치 편의성:** 서버(Ubuntu + Conda 환경)에 C++ 의존성 문제 없이 쉽게 설치 가능해야 함.
3. **토큰 분해 깊이:** 어절 단위가 아니라, 의미를 다채롭게 담고 있는 실질 형태소/형식 형태소 단위로 잘게 쪼개주어야 "다양성"을 민감하게 측정할 수 있음.
4. **신조어/위키 데이터 대응:** 나무위키 기반 데이터를 쓸 것이므로 신조어 인식률이 높아야 함.

## 3. 추천안: **Kiwi (Kiwipiepy)**

우리 연구의 목적상 "수만~수십만 문장의 텍스트를 빠르게 읽고 형태소 품사(POS) 집합을 뽑아내는 것"이 목표입니다. 

**Kiwi를 추천하는 강력한 이유:**
1. **서버 세팅의 쾌적함:** 리눅스 서버에서 Python 3.11 환경에 `pip install kiwipiepy` 한 줄이면 C++ 컴파일 에러 없이 즉시 동작합니다. (Mecab은 가끔 꼬이면 하루종일 고생합니다)
2. **탁월한 속도:** 멀티스레드를 기본 지원하여 대용량 코퍼스를 스캐닝하는 데 최상의 성능을 냅니다.
3. **나무위키/웹 데이터 특화:** 신조어 추출 능력이 뛰어나서 우리가 구축하려는 데이터셋에 잘 맞습니다.

### 실험에서의 활용 방안 (다양성 점수 측정 로직 예시)

```python
from kiwipiepy import Kiwi

kiwi = Kiwi(num_workers=4) # 멀티코어 활용
text = "양자화 라이브러리를 사용해봅시다."
tokens = kiwi.tokenize(text)

# 형태소와 품사(POS)의 조합을 하나의 '특징'으로 간주
morph_features = set([(token.form, token.tag) for token in tokens])

print(len(morph_features)) # 이 길이가 길수록 '형태소 다양성 점수'가 높음
```

## 4. 결론 및 향후 계획

사용자와의 논의 결과:
1. **Kiwi(kiwipiepy)**를 **1순위(Primary)** 형태소 분석기로 채택합니다. 원활한 환경 세팅과 빠른 속도로 실험 사이클을 단축하기 위함입니다.
2. 단, 기존 NLP 논문들에서 표준으로 많이 쓰이는 **Mecab**도 **염두에 두고(후순위 대안)**, 나중에 논문 리뷰어의 지적이나 비교 실험이 필요할 경우를 대비해 스크립트 작성 시 분석기를 추상화(Interface)하여 쉽게 교체할 수 있도록 모듈화합니다.
