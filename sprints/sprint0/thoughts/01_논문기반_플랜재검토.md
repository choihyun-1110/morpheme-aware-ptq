## 2. 논문 분석에서 발견한 Planning 관련 인사이트

### 🔴 GPTQ vs AWQ의 calibration 민감도 차이 → 우리 연구의 핵심 변수

사용자 노트에서 가장 중요한 발견:

- **GPTQ**: Calibration data에 맞춰 가중치를 **수정(Update)**, 특정 데이터에 과적합 가능
  - AWQ 논문 노트에서: *"GPTQ는 보정 데이터셋에 과적합되어, 전혀 다른 데이터가 들어오면 성능이 떨어질 수 있다"*
- **AWQ**: 가중치 값을 수정하지 않고, **채널별 평균 활성화 크기(scale)**만 사용
  - *"특정 데이터셋의 내용에 깊게 의존하지 않아 일반화 성능이 좋다"*

**→ 이것이 우리 연구에 주는 시사점:**
- 한국어 형태소 다양성 calibration의 효과는 **GPTQ에서 더 크게** 나타날 가능성
- AWQ는 calibration 내용에 덜 민감하므로, 조건 A/B/C 간 차이가 **작을 수 있음**
- **실험 설계에 반영 필요:** GPTQ에서 효과가 크고 AWQ에서 작으면, "calibration 방법론(regression vs scaling)에 따른 민감도 차이"라는 추가 insight 도출 가능

### 🟡 DuQuant의 Future Work = 우리 연구

DuQuant 논문(NeurIPS 2024 Oral)의 Limitation 섹션:
> *"현재는 WikiText2에서 무작위 샘플링을 하는데, 더 정교한 데이터 선택 전략이 있다면 성능을 더 끌어올릴 수 있을 것"*

**→ 우리가 정확히 이걸 한국어 맥락에서 하려는 것!** planning.md에 이 인용을 추가하면 연구 동기가 더 강화됨.

### 🟡 QDrop의 flatness 이론 → planning.md 가설 메커니즘 보강

사용자의 QDrop 노트에서:
- Calibration 시 activation 양자화를 어떻게 처리하냐에 따라 **flatness(평탄도)**가 달라짐
- 평탄한 모델 = 일반화 성능 우수
- **→ 우리 가설:** 형태소 다양성 높은 calibration → activation distribution의 더 넓은 coverage → 더 평탄한 양자화 결과?

### 🟢 ZeroQuant의 Token-wise dynamic range

ZeroQuant에서 확인된 사실:
- *"토큰마다 activation의 dynamic range가 극단적으로 다를 수 있음"*
- **→ 한국어의 높은 subword fragmentation** → 토큰 수가 많아짐 → activation distribution이 더 다양? → calibration에서 이를 반영하면 양자화 품질 향상 가능?

## 3. GPU 제약을 감안한 Planning 재검토

### 실험 가능성 평가

| 항목 | planning.md 원안 | GPU 제약 반영 수정안 |

| 모델 수 | 3개 | **2개** (EXAONE-7.8B + Llama-3.1-8B) |

planning.md에 없는데 넣으면 좋을 것:
- GPTQ가 calibration에 민감 → 한국어 calibration 효과가 클 수 있음
- AWQ가 calibration에 둔감 → 효과가 작으면 "AWQ의 robustness" 확인
- **이 차이 자체가 좋은 분석 포인트** (Case 4의 확장)
