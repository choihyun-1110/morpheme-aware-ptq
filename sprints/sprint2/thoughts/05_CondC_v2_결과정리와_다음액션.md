## 1. 배경

기존 KoBEST 1차 결과는 다음과 같았다.

- Cond A: `0.5981`
- Cond B: `0.6176`
- Cond C: `0.6062`

기존 C는 A보다 좋았지만 B보다 낮았고, 특히 calibration set이 너무 짧은 문구/표제어형 샘플을 많이 포함한다는 문제가 제기되었다. 이를 보정하기 위해 C 알고리즘에 문장형 필터, 길이 보너스, 문장 종결 보너스를 추가한 `Cond C_v2`를 만들었다.

## 4. 현재 해석

이번 결과는 "짧은 문구를 줄이면 무조건 좋아진다"는 가정을 지지하지 않는다.

오히려 현재 개정안은 다음 부작용을 냈을 가능성이 있다.

- 문장형 필터가 너무 강해져 calibration set 분포가 좁아짐
- 길이 보너스가 diversity objective를 과도하게 눌렀음
- 자연스러운 문장을 더 많이 뽑았지만, 질문-응답형 이해에 필요한 표현 다양성이나 분포 폭이 줄어들었을 수 있음

특히 BoolQ 하락폭이 큰 점은, 단순히 "문장다움"을 올리는 것만으로는 calibration 품질이 좋아지지 않음을 시사한다.

## 5. 연구적으로 지금 의미하는 것

현재까지의 메시지는 비교적 명확하다.

- 영어 랜덤(A)보다 한국어 랜덤(B)이 강하다.
- 형태소 다양성 기반 C는 A보다 낫지만, B를 넘지 못했다.
- 문장성 보정까지 추가한 C_v2는 오히려 성능이 떨어졌다.

따라서 현재 시점에서 가장 안정적인 주결론은 다음이다.

> "현재 구현된 morphology-aware objective는 일부 개선 신호는 보이지만, 아직 한국어 랜덤 baseline(B)을 안정적으로 이길 정도로 성숙하지 않았다."

## 6. 다음 액션

지금 가장 먼저 해야 할 일은 추가 튜닝이 아니라 원인 분해다.

1. `Cond B / Cond C / Cond C_v2`의 calibration set 통계를 다시 비교
   - 평균 어절 수
   - 평균 subword token 수
   - 문장 종결형 비율
   - 총 고유 형태소 커버리지
   - 총 고유 subword 커버리지

2. 왜 BoolQ가 특히 크게 떨어졌는지 가설 정리
   - 문장 길이 문제인지
   - 문장형 필터로 인해 표현 폭이 줄었는지
   - coverage bonus 대비 length bonus 비중이 과했는지

3. C 알고리즘 추가 수정은 잠시 보류
   - 지금 단계에서 바로 `beta`, `gamma`를 다시 튜닝하면 탐색이 산으로 갈 가능성이 큼
   - 먼저 어떤 통계량이 실제 성능 하락과 연결되는지 확인해야 함

4. 메인 실험 축은 `A / B / C`로 유지한 채 `kmmlu` 추가 여부 결정
   - 연구 본선 비교는 현재로서는 `B > C > A` 해석이 더 안정적임
   - C_v2는 실패 사례로 남기되, 본선 후보로 바로 올리지는 않는 편이 타당함

## 7. 현재 권장 판단

지금 시점의 우선순위는 다음과 같다.

1. `B / C / C_v2` calibration 통계 비교 문서 작성
2. `A / B / C`를 메인 비교군으로 두고 `kmmlu` 실험 확장 여부 결정
3. 그 다음에만 C objective 재설계 착수
