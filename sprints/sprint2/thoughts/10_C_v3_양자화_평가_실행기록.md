## 4. 결과 해석

사전에 정의한 해석 기준 대비:

| 시나리오 | 기준 | 실제 결과 |
|---------|------|----------|
| kobest avg > 0.6176 (B 초과) | 형태소 다양성 가설 강력 지지 | ✅ **0.6354 (B+0.018)** |
| BoolQ ≥ 0.77 | 비한국어 제거 효과 유효 | ✅ **0.8077** |

**가설 검증:** "비한국어 혼합 문장이 C의 SFS를 인위적으로 높이고 calibration 품질을 저하시켰다"는 진단이 맞았음.

주요 관찰:
- **BoolQ +0.051** (C 대비): 가장 큰 수혜 태스크. 비한국어 제거 후 한국어 이해력 관련 태스크에서 회복
- **WiC 0.5302**: FP16(0.5008) 초과 — 양자화 후에도 원본보다 높은 수치 (통계적 노이즈 가능성 있으나 주목할 만함)
- **SentiNeg 소폭 하락** (B=0.6171 → C_v3=0.6322, B 대비는 상승): B의 SentiNeg 강점은 C_v3도 어느 정도 유지
- **단일 런**이므로 B vs C_v3 차이(0.018)가 노이즈인지 신호인지는 여전히 불확실 — 재현 실험 필요

## 5. 트러블슈팅: CUDA OOM (eval 최초 실행 시)

첫 eval 수동 시도에서 `RuntimeError: CUDA error: CUBLAS_STATUS_ALLOC_FAILED` 에러 발생.

**원인:** 자동 eval 연동 체인(`docker exec -d`)이 이미 lm_eval을 시작한 상태에서
터미널에서 한 번 더 실행 → 두 프로세스가 동시에 GPU 0에 모델 로드 → 24GB VRAM 초과.

**조치:** 중복 프로세스(PID 14592) kill → 남은 프로세스(14521)가 정상 완료.

**교훈:** `docker exec -d`로 자동 연동 체인을 걸어둔 경우, 수동 재실행 전에 반드시 `pgrep -f lm_eval` 확인.
