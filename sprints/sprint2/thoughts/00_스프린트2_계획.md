# Sprint 2 기획안: 본격적인 양자화 성능 검증

**작성일시**: 2026-03-17
**목표**: Sprint 1에서 뽑은 데이터를 바탕으로 **"한국어 형태소 다양성을 높인 캘리브레이션 셋이 진짜 PTQ 성능(제로샷)을 높이는가?"**를 실증한다.

## 1. 평가 기준 (Metrics)
우리의 궁극적 타겟은 한국어 성능 하락 방어입니다. `lm-evaluation-harness` (polyglot-ko 포크 등 호환 버전)를 사용하여 다음 3가지를 중점적으로 봅니다.
- **Ko-HellaSwag**: 문맥 추론 (가장 기본적인 한국어 자연스러움)
- **Ko-MMLU**: 전반적인 지식 (양자화로 인해 지식이 날아갔는지 확인)
- **Ko-TruthfulQA** (옵션): 환각 (Outlier 파괴로 인해 모델이 헛소리를 늘어놓기 시작하는지)

> **비교군**:
> 1. 원본 FP16 모델 (Baseline 0 - 상한선)
> 2. 조건 A (영어 랜덤) 4bit 양자화 모델
> 3. 조건 B (한국어 랜덤) 4bit 양자화 모델
> 4. 조건 C (우리 알고리즘) 4bit 양자화 모델

## 2. 양자화 조건 설정 (AutoGPTQ)
- **방식**: GPTQ 
- **Bit**: 4-bit (가장 하락의 골이 체감되는 대중적 비트)
- **Group Size**: 128
- **Act Order**: `True` (outlier 보존에 좋으나, 추론 속도가 살짝 떨어짐. 보통 논문에서 성능 최대화를 위해 켬)

## 3. S2-1 ~ S2-2 아키텍처 (파이프라인)
```
[입력] calibration_set_X.json
   ↓ parsing
List[str] (텍스트 문장 128개)
   ↓ AutoGPTQ Tokenization
[HuggingFace Dataset 포맷]
   ↓ 
quantize() 실행
   ↓
[저장] results/quantized_models/SOLAR_10.7B_4bit_condition_X/
   ↓
[평가] lm-eval --model hf --model_args pretrained=...
   ↓
[결과] JSON 메트릭 출력
```

## 4. 리스크 및 고민 지점
1. **LM-Eval-Harness 한국어 설정**: 공식 `lm-evaluation-harness`에 한국어 벤치마크가 전부 있는지, 아니면 eleutherAI 포크나 다른 래퍼 코드를 써야 하는지 확인 필요.
2. **평가 소요 시간**: SOLAR 10.7B를 전부 평가 돌리려면 꽤 시간이 걸리기 때문에, S2-3 단계에서 백그라운드 스크립트로 던져버리고 우리는 병렬로 다른 작업(S2-4 데이터 시각화 준비)을 해야 함.

## 5. 구현 업데이트 메모 (2026-03-17)
- 현재 서버 운영 방식은 호스트 직접 실행이 아니라, `llm-dev` Docker 컨테이너 안 `llm-quant` conda 환경을 기준으로 삼는 것이 맞음.
- 따라서 `src/run_eval.sh`는 다음 흐름을 따르도록 수정됨.
  1. 로컬에 `lm_eval`가 있으면 그대로 사용
  2. 없으면 `docker exec llm-dev ...`로 재실행
  3. 컨테이너 안에서 `conda activate llm-quant` 수행 후 평가
- 초기 계획 문서에는 Ko-HellaSwag, Ko-MMLU를 적어두었지만, 현재 로컬 harness 등록 태스크명과 Sprint 0 평가 프레임워크 문서를 기준으로 1차 평가는 `kobest` 그룹을 우선 사용하도록 정리함.
- 양자화 모델 로딩 시 기본 Hugging Face 로더가 `transformers`의 GPTQ 경로를 타면서 `optimum.gptq`를 요구하는 문제가 있었음.
- 그러나 현재 양자화 산출물은 `src/run_quant.py`에서 `AutoGPTQForCausalLM`으로 저장한 GPTQ 체크포인트이므로, 평가 시에도 `autogptq=<모델 safetensors>`를 명시해 같은 계열 로더를 사용하도록 수정함.
- 즉, 지금 평가 파이프라인의 실제 형태는 아래와 같음.

```
[호스트] bash src/run_eval.sh ...
   ↓ lm_eval 없으면 자동 위임
[Docker: llm-dev]
   ↓ conda activate llm-quant
lm_eval --model hf --model_args pretrained=...,autogptq=...
   ↓
KoBEST 평가 수행
```
