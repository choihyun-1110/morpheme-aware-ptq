## 1. 지금 잘못하고 있는 것들

### 1-1. 파라미터 튜닝 루프에 갇혀 있음

C → C_v2 진행이 전형적인 "결과가 나쁘면 파라미터 바꿔보기" 패턴이다.
근거는 "짧은 문구가 많으니까 필터 추가하자"였는데, 결과는 더 나빠졌다.

문서에도 "C 알고리즘 수정 보류, 원인 분해 먼저"라고 쓰여 있는데 이 판단은 맞다.
그런데 그 전에 이미 C_v2를 실행해버린 게 순서가 거꾸로였다.

### 1-2. 가설과 실험 사이의 연결고리가 검증된 적 없음

연구 가설의 논리 구조:

```
형태소 다양성 ↑  →  activation 다양성 ↑  →  GPTQ calibration 품질 ↑
```

지금까지 측정한 건 맨 앞(UMR, TTR, SFS)과 맨 뒤(KoBEST 점수)뿐이다.
**중간 링크인 activation 다양성을 한 번도 확인한 적이 없다.**
C가 B보다 정말로 더 다양한 activation을 유도하는지 모른 채 결과만 보고 있는 것이다.

### 1-3. FP16 베이스라인이 없음

계획 문서에 "Baseline 0 - 상한선"으로 적혀 있는데, 실제로 실행한 적이 없다.
이게 빠지면 각 조건이 원본 FP16 대비 **얼마나 잃었는지**를 모르기 때문에,
지금 비교는 "덜 나쁜 것"을 찾는 건지 "실제로 좋은 것"을 찾는 건지 알 수 없다.

### 1-4. 단일 런에 해석이 너무 많음

C(0.6062) vs B(0.6176) 차이가 0.0114인데, 단일 실험이다.
가설 1~5 같은 해석들은 사실 0.0114 차이 하나에 너무 많은 이야기를 얹고 있다.
C 자체가 B보다 진짜로 나쁜지조차 아직 통계적으로 확인된 것이 없다.

### 1-5. B가 왜 좋은지를 이해하지 못한 채 C만 파고 있음

B가 강한 이유 후보들:

- **길이** (평균 74.73 토큰 vs C의 31.47)
- **자연 분포** (랜덤 샘플이라 분포 편향 없음)
- **형태소 커버리지 총량** (1620 vs 1130)

이 세 가지 중 무엇이 진짜 원인인지 분리한 실험이 없다.
예를 들어, "길이만 B 수준으로 맞춘 랜덤 선택" vs "C 알고리즘으로 선택한 것"을 비교하면
길이 효과를 분리할 수 있는데, 그런 제어 실험이 없다.

## 2. 실제로 해야 할 것 (흐름 기반)

우선순위 문서에 쓰인 것 말고, 흐름상 가장 막혀 있는 병목 기준:

### 1순위: FP16 베이스라인 실행

지금 당장 해야 한다.
이게 없으면 논문이든 결론이든 "상대적 순위"만 말할 수 있고,
"얼마나 보존했냐"는 절대값을 말할 수 없다.

### 2순위: calibration set 통계 비교 (B vs C vs C_v2)

핵심 확인 항목:
- C_v2가 C보다 평균 토큰 수가 길어졌는가?
- 길어졌어도 B보다 짧은가?
- 고유 형태소 커버리지는 어떻게 달라졌는가?

이게 확인되면 "길이 효과"인지 "다양성 효과"인지 판단 근거가 생긴다.

### 3순위: kmmlu는 지금 안 해도 됨

KoBEST에서 B>C>A 패턴 하나도 아직 불안정한 상황에서
kmmlu를 추가하면 해석해야 할 결과만 늘어난다.
kmmlu는 주결론이 안정된 다음에 검증용으로 쓰는 것이 맞다.

### 4순위: C 알고리즘 재설계는 B가 왜 강한지 알기 전에 하면 안 됨

지금 단계에서 alpha/beta/gamma를 다시 건드리는 것은
아직 이해 못한 변수 공간을 랜덤하게 탐색하는 것과 같다.
